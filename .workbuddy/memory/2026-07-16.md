## 菜单拖拽排序 - 修复「拖拽不了」
- 现象：加了拖拽范围高亮（rowClassName 读 dragParentId 响应式）后，拖拽直接失效。
- 根因：在 `dragstart` 里改响应式状态会触发 antd-vue 表格重渲染，浏览器在执行 drag 前 patch 了被拖的 tr，原生 HTML5 拖拽被取消。
- 修复：拖拽全过程改为零 Vue 重渲染——拖拽上下文用普通变量（非 ref）；行高亮/提示条直接操作 DOM。customRow 返回里带 `data-menu-id` 便于按 id 定位 tr；dragstart 遍历 tbody tr 按父级比对加 class（drag-row-current/sibling/disabled），dragend/drop 清除。顶部提示条默认 display:none，JS 控制显隐。
- 结论（可复用）：antd-vue 树形表格做 HTML5 行拖拽时，千万不要在 dragstart 里改会触发表格重渲染的响应式状态；用普通变量 + 直接 DOM 操作。

## 菜单/角色拖拽排序 - 最终修复（手柄方案，验证通过）
- 根因（真正）：前几版（含"纯 DOM 版"）仍依赖 `customRow` 给整行 tr 设 `draggable:true` + `onDragstart`。在 antd-vue 4.2.6 树形表格下，整行 draggable 实测**拖不动**（用户多次反馈"不能拖拽排序"）。antd BodyRow 把 customRow 属性 spread 到 tr，但整行 draggable 在树形场景下不生效/被内部交互吞掉。
- 最终方案（menu 与 role 一致）：
  1. 拖拽源 = 「拖拽」列的六点图标 `<div draggable="true" @dragstart="onDragStart($event, record)">`（明确、浏览器原生支持，不依赖 antd 内部）。

## 菜单/角色拖拽排序 - 最终根治（SortableJS，真实浏览器验证通过）
- 根因（最终确认）：前面所有 HTML5 方案（customRow 整行 draggable、手柄 draggable+容器 @drop）在 antd-vue 4.2.6 表格下**都拖不动**，用户反复反馈"拖拽不了"即此。Playwright 合成 DragEvent 会假阳性通过，必须用真实鼠标才能验出失败。
- 方案：改用 `sortablejs`（基于 pointer 事件）。`pnpm add sortablejs`（依赖已入 package.json）。
- 菜单页 `src/views/system/menu/index.vue`：
  - 外层 `<div ref="tableWrapRef">` 包住 ATable；`initSortable()` 取 `root.querySelector('.ant-table-tbody')`（注意：ref 不能放 ATable 上，那拿的是组件实例）；`new Sortable(tbody, { filter: 按钮/开关/展开箭头/input/a, animation:150, onMove, onEnd })`。
  - 整行可拖（交互元素除外），降低抓取难度；`:deep` 配 sortable-ghost/chosen/drag 高亮，行 `cursor:grab`。
  - 同级约束：`onMove` 比对 parentId 返回 false 阻止跨级；`onEnd` 按 DOM 顺序重排 + `fetchMenuSort({parentId,ids})`。
  - 跨级拦截提示：普通变量 `dragTargetParentId` 在 onMove 记录目标 parentId，onEnd 位置未变且不同级时 `warning('仅支持同级菜单排序')`（onEnd.evt.related 完全拦截时为 null 不可靠）。
  - `watch([tableData, expandedKeys], () => nextTick(initSortable))` 重建实例。
- 角色页 `src/views/system/role/index.vue`：同样改 SortableJS（原也是失效的 HTML5 方案），整行可拖，onEnd 重排当前页 + `fetchRoleSort({startOrder,ids})`。
- 真实浏览器验证（Playwright + 系统 Chrome + page.mouse 真实拖拽 + 真实后端写回）：
  - 菜单同级（id4→id7）：顺序改变，「排序已更新」✅；跨级（id1→id3）：行不动 + 提示「仅支持同级菜单排序」✅。
  - 角色（id1行→底部）：顺序 `["1","2","3"]`→`["2","3","1"]`，「排序已更新」✅。
- 经验：antd-vue 表格行拖拽一律用 sortablejs；验证必须用真实鼠标；合成 DragEvent 不可信。已同步更新 MEMORY.md「拖拽排序实现要点」。

## 修复：部门管理跨级拖拽不弹「仅支持同级」提示（BUG）
- 现象：菜单管理跨级拖拽弹「仅支持同级菜单排序」，部门管理跨级却无任何提示。
- 根因：`src/views/system/dept/index.vue` 的 Sortable `onMove` 只在**同级**时才设置 `dragTargetParentId`；跨级时 `return false` 但未记录目标父级，导致 `onEnd` 里 `dragTargetParentId` 仍为 `-1`，warning 条件 `dragTargetParentId !== -1 && ...` 不满足 → 静默失败，不弹提示。
- 修复：让 `onMove` 无论是否同级都记录 `dragTargetParentId = related.parentId ?? 0`（与菜单管理 `index.vue` 的 onMove 对齐），return 值仍按同级判定。这样跨级时 onEnd 能检测到 `dragTargetParentId !== dragged.parentId` 并弹「仅支持同级部门排序」。
