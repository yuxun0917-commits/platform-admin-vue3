## 日期选择器中文显示修复（pnpm-workspace.yaml）
- 用户管理新增/编辑弹窗的出生日期选择器显示 "2028年 Oct"、星期为英文
- 根因：项目依赖 dayjs 1.11.19，antd-vue 自身解析到 dayjs 1.11.21，pnpm 安装了两个版本；`dayjs/locale/zh-cn` 只注册在应用实例上，antd-vue 的 DatePicker 调用的是另一个 dayjs 实例，读不到中文月/星期
- 修复：在 `pnpm-workspace.yaml` 加 `overrides: dayjs: 1.11.19`，强制所有包共用同一版本，使 antd-vue 的 DatePicker 与应用的 dayjs 共享 locale 数据
- 重新执行 `pnpm install` 后，antd-vue 的 `node_modules/dayjs` 已指向 `dayjs@1.11.19`
- 需重启 dev server 让 Vite 重新解析模块生效
