## 修复 vite-env.d.ts 的 no-shadow lint 错误（commit 被拦截）
- 现象：`pnpm commit` 在 lint-staged 的 `eslint --fix` 阶段失败，唯一 error 为 `src/typings/vite-env.d.ts` 第 11 行 `'ImportMeta' is already declared in the upper scope on line 119`（`@typescript-eslint/no-shadow`）。`.env` 那条 `File ignored` 仅 warning，不影响。
- 根因：`declare namespace Env` 内部原本是 `interface ImportMeta extends ImportMetaEnv {`（顶层第 119 行 `interface ImportMeta { readonly env: Env.ImportMeta }`）。`ImportMeta` 在 namespace 内被判定遮蔽全局 `ImportMeta`，触发 no-shadow。该文件此前一直未被 staged，本次因改动 `VITE_SERVICE_SECOND_FACTOR_CODES` 首次进入 lint 范围才暴露。
- **正确修复（关键坑）**：改用 Soybean 标准写法——把环境字段接口放到**全局作用域**命名为 `ImportMetaEnv`（与 vite/client 的全局 `ImportMetaEnv` 合并，从而同时拥有 `VITE_*` 与 `PROD`/`DEV`/`MODE`），`import.meta.env` 也指向全局 `ImportMetaEnv`。注意：
  - 若只在 namespace 内改名 `interface ImportMetaEnv extends ImportMetaEnv`，`extends` 右侧会被解析成**自身**（局部声明），导致 `Env.ImportMetaEnv` 丢失 `PROD`/`DEV` → 后续 `vue-tsc` 报 `Property 'PROD'/'DEV' does not exist on type 'ImportMetaEnv'`（4 处：app.ts:17、request/index.ts:10、theme/index.ts:159、theme/shared.ts:15）。
