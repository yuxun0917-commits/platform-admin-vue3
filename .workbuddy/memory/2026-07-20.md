## git 提交故障修复（2026-07-20 补充）
- `pnpm commit`（`sa git-commit`）失败排查：
  - 现象：`vue-tsc` 跑完后 `lint-staged could not find any staged files` → `git commit` exit 1，且 `working tree clean`。根因是上一次失败提交时全部改动被 `stash@{0}: lint-staged automatic backup` 收走。恢复：`git stash apply stash@{0}`（保留备份）/ `git stash pop`（删备份）。
  - `sa git-commit` 脚本流程 = prompt → vue-tsc → lint-staged → `git commit`，**不会自动 `git add`**。提交前必须自己先 `git add`（否则 lint-staged 无 staged 文件、commit 无可提交内容而失败）。本次最终用 `git add -A && git commit -m "feat(other): 分批提交"` 走通（pre-commit 钩子照常跑 lint-staged + vue-tsc）。
- elegant-router 生成文件被清空导致 vue-tsc 全崩：
  - 现象：vue-tsc 报 `Cannot find module '@elegant-router/types'`、`File '.../transform.ts' is not a module`，并连锁大量 `implicitly has an 'any' type` 假错误。根因：`src/typings/elegant-router.d.ts`（315 行）与 `src/router/elegant/transform.ts`（223 行）两个 elegant-router **生成文件被清空成 0 字节**（被 stash 一并带走）。
  - 修复：`git checkout HEAD -- src/typings/elegant-router.d.ts src/router/elegant/transform.ts` 恢复；因业务已把 `home` 路由改 `dashboard`，需同步把 `.d.ts` 的 RouteMap（`"home":"/home"`→`"dashboard":"/dashboard"`）、`FirstLevelRouteKey`/`LastLevelRouteKey` 里的 `"home"`，以及 `transform.ts` 内 `routeMap` 常量的 `"home":"/home"` 一并改成 `dashboard`。改完 vue-tsc 立即全过。
  - 真正重建生成文件需触发 vite 插件（`pnpm dev`/`pnpm build`）；`pnpm gen-route`（`sa gen-route`）是**交互式新建单条路由**命令（会卡在 `please enter route name` 提示），不是重建全部。
  - 定时任务日志抽屉（src/views/monitor/job/modules/job-log-drawer.vue）：首列「日志编号」(dataIndex:id) 改为「序号」(key:index, width:70)，bodyCell 里按分页计算 `((current-1)*pageSize + index + 1)` 渲染连续序号。
